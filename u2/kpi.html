Unidad 2 · Tecnologías Habilitadoras Digitales (THD)

RA2. Caracteriza las tecnologías habilitadoras digitales necesarias para la adecuación/transformación de las empresas a entornos digitales describiendo sus características y aplicaciones.

Criterios de evaluación

2. Caracteriza las tecnologías habilitadoras digitales necesarias para la adecuación/transformación de las empresas a entornos digitales describiendo sus características y aplicaciones.

  1. Se han identificado las principales tecnologías habilitadoras digitales.
  2. Se han relacionado las THD con el desarrollo de productos y servicios.
  3. Se ha relacionado la importancia de las THD con la economía sostenible y eficiente.
  4. Se han identificado nuevos mercados generados por las THD.
  5. Se ha analizado la implicación de THD tanto en la parte de negocio como en la parte de planta.
  6. Se han identificado las mejoras producidas debido a la implantación de las tecnologías habilitadoras en relación con los entornos IT y OT.
  7. Se ha elaborado un informe que relacione las tecnologías con sus características y áreas de aplicación.

Tareas relacionadas

1. Inventario de THD (a) — qué es y cómo hacerlo

Elabora una lista de 10–12 THD. Para cada una incluye: definición ≤30 palabras, 3 características y 1 ejemplo real del sector. Cita 1 fuente reciente.

Evidencia: tabla en index.html Commit: feat: inventario THD

2. Casos de uso (b) — del laboratorio al cliente

Vincula cada THD con una función de producto/servicio (qué mejora exactamente). Añade un mini-roadmap: MVP → beta → versión estable, con 1 riesgo y 1 mitigación.

Evidencia: sección “Casos de uso” KPI: NPS, tiempo de ciclo

3. Sostenibilidad (c) — medir impacto

Define 2 KPI (p. ej., kWh/u, CO₂eq) y muestra “antes vs después” en una tabla simple. Explica en 2 frases cómo la THD reduce consumo o residuos.

Evidencia: tabla KPI Commit: feat: bloque sostenibilidad

4. Nuevos mercados (d) — dónde vender

Identifica 2 oportunidades por THD (XaaS, plataforma, datos como producto, etc.). Añade un ejemplo real breve del sector (1–2 líneas).

Evidencia: lista con bullets KPI: ingresos nuevos %

5. Negocio vs planta (e) — el dibujo que lo aclara

Crea un esquema IT/OT (negocio ↔ planta) con 4 cajas: ventas, logística, producción, mantenimiento. Pon la THD encima de dónde actúa y qué dato genera.

Evidencia: imagen/diagrama Commit: feat: esquema it-ot

6. Mejoras IT/OT (f) — números a la vista

Compara 3 KPI técnicos antes/después (p. ej., OEE, MTBF, MTTR). Usa flechas ↑ ↓ y un comentario de una línea por cada mejora.

Evidencia: tabla “antes/después” Meta: OEE +5 p.p.

7. Informe final (g) — todo en una sola página

Integra todo en una matriz THD → características → aplicaciones → KPI y escribe 3 conclusiones + 2 próximos pasos. (Si haces vídeo, solo captura de pantalla).

Evidencia: index.html publicado Commit final: docs: informe RA2

Repositorio y publicación

Flujo mínimo de control de versiones (Git) + GitHub Pages:

# 1) Clonar
git clone https://github.com/<usuario>/ejercicio_RA2_THD
cd ejercicio_RA2_THD

# 2) Trabajo iterativo (a → g)
git add .
git commit -m "feat: inventario inicial de THD (a)"
git push origin main

# 3) Publicar con GitHub Pages
# Settings → Pages → Branch: main / root

Privacidad: si incluyes vídeo, usa solo captura de pantalla (sin rostro) y compártelo como no listado o solo en Moodle.

Checklist de entrega (4 ítems)
