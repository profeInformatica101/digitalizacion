Tarea 2 · 1º DAW · Digitalización, IHM (pasado–futuro), IA y GitHub

UD1 · Digitalización aplicada a los sectores productivos (GS · Módulo 1665)

Para Moodle

Objetivo

  • Comparar la Interacción Hombre–Máquina (IHM) del pasado y del futuro y relacionarla con la digitalización en la empresa.
  • Reflexionar sobre cómo la IHM y la IA cambian la organización y la toma de decisiones.
  • Practicar GitHub (fork, clone, commit, push) sobre una web estática.
  • Construir un prototipo web (HTML, CSS, JS) con criterios de calidad básicos.

Vídeos y preguntas (responder en un documento)

Vídeo 1 · IHM – Pasado: IBM 029 Keypunch (1964)

  1. Explica brevemente cómo interactúa la persona con la máquina en el vídeo (entradas, salidas, errores).
  2. ¿Qué limitaciones ves para usar este tipo de interacción en una empresa actual?
  3. Relaciona esta forma de trabajar con el concepto de digitalización: ¿qué datos se capturan y cómo afectan a la organización? (RA1-a,b)

Vídeo 2 · IHM – Futuro: VOX 114 (The Time Machine, 2002)

  1. Describe la interacción con VOX 114: ¿qué tipo de comandos, respuestas y ayudas ofrece?
  2. Indica 2 ventajas y 2 riesgos de usar un asistente tipo IA en una empresa.
  3. Explica cómo un sistema así podría integrarse con IT (negocio) y OT (operación) para mejorar decisiones en tiempo real. (RA1-b,c)
Entrega de esta parte: Responde a las preguntas en un PDF (1–2 págs) o DOCX y adjúntalo en Moodle.

Prototipo web (fork de mi-web-estatica) con otros criterios

Realiza un fork de github.com/profeInformatica101/mi-web-estatica y crea un prototipo que represente una empresa ficticia que usa IA.

Requisitos mínimos del prototipo

  • Estructura: index.html, style.css, script.js.
  • Contenido: nombre de la empresa, propuesta de valor y una sección “IA en acción”.
  • Accesibilidad básica: etiquetas semánticas (header, main, footer), texto alternativo en imágenes y contraste legible.
  • Responsive: diseño que se adapte a móvil (mínimo una regla @media o clases de Bootstrap si las integras).
  • JS sencillo: botón o formulario con interacción básica (alerta/validación/contador).

Evidencias en GitHub

  1. Al menos 2 commits con mensajes claros (feat:, fix:, style:).
  2. Capturas de terminal o historial de commits mostrando git clone, git add, git commit, git push.
  3. (Opcional) Activar GitHub Pages para publicar el prototipo.
Entrega del prototipo: sube un ZIP con la carpeta del proyecto o pega el enlace a tu fork en GitHub.

Rúbrica (6 puntos) · RA1 (Módulo 1665)

En esta tarea NO se evalúan todos los criterios del RA1. Solo: (a), (b), (c) y (g).

Criterio 3 – Excelente 2 – Notable 1 – Básico Peso
(a) Concepto de digitalización Definición clara y breve + 1 ejemplo. Definición correcta. Definición incompleta/ambigua. 1.0
(b) Impacto organizativo Relaciona con cultura/procesos y aporta ejemplo. Relación parcial (solo cultura o procesos). Relación superficial. 1.5
(c) IT vs OT Explica ≥3 diferencias y ≥2 similitudes con ejemplo. Explicación correcta pero incompleta. Confusión o superficial. 1.5
(g) Ventajas extremo a extremo ≥3 ventajas claras (costes, trazabilidad, calidad, sostenibilidad). 2 ventajas claras. 1 ventaja vaga. 2.0
Total Suma ponderada 6.0

Mínimo para superar: 3.0. En Moodle: rúbrica 1–3 por criterio.

Formato de entrega en Moodle

  • Documento con respuestas a los vídeos: PDF (1–2 págs) o DOCX.
  • Prototipo: ZIP o enlace al fork en GitHub (con capturas de evidencias).
  • Nombrado: UD1_DAW1_T2_ApellidosNombre.pdf y UD1_DAW1_T2_Prototipo.zip (si aplica).

Sugerencia: exporta a PDF con Ctrl/Cmd + P → “Guardar como PDF”.
