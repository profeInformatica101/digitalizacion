Conceptos base

8 términos
Inteligencia Artificial
Conceptos base
▾
Conjunto de técnicas y sistemas capaces de realizar tareas que requieren capacidades cognitivas humanas: clasificar, predecir, recomendar, reconocer patrones, generar contenido o tomar decisiones de forma autónoma.
Ejemplos
  • Asistentes virtuales (Siri, Alexa, Google Assistant)
  • Filtros anti-spam y detección de fraude bancario
  • Sistemas de recomendación (Netflix, Spotify)
  • IA generativa: texto, imagen y código (ChatGPT, Copilot)
CriteriosCE-aCE-cCE-f
IA Débil (Narrow AI)
Conceptos base
▾
IA diseñada para realizar una tarea específica y acotada. No generaliza más allá de su dominio de entrenamiento. Es la única forma de IA existente hoy en todos los sistemas comerciales.
Ejemplos
  • Detección de spam en correo electrónico
  • Reconocimiento facial en smartphones
  • Motor de recomendación de productos en e-commerce
  • Traducción automática (DeepL, Google Translate)
CriteriosCE-aCE-d
IA Fuerte (AGI)
Conceptos base
▾
Concepto hipotético de una IA con capacidades cognitivas generales comparables o superiores a las humanas. Capaz de aprender y razonar en cualquier dominio. No existe operativamente en la actualidad.
Contexto

Es un objetivo teórico de la investigación en IA que genera debate ético sobre riesgos existenciales. Los LLMs actuales (GPT-4, Gemini) son IA Débil muy capaz, no IA Fuerte.

CriteriosCE-c
IA Simbólica
Conceptos base
▾
Enfoque de IA basado en representación explícita del conocimiento mediante reglas, lógica y símbolos. El razonamiento sigue cadenas de inferencia predefinidas por expertos humanos. Interpretable pero difícil de escalar.
Ejemplos
  • Sistemas expertos de diagnóstico médico (MYCIN)
  • Motores de reglas de negocio en banca
  • Chatbots con flujos predefinidos basados en menús
CriteriosCE-a
IA Subsimbólica
Conceptos base
▾
Enfoque basado en el aprendizaje automático a partir de datos, sin representar el conocimiento de forma explícita. Los modelos extraen patrones estadísticos. Incluye ML, Deep Learning y LLMs.
Ejemplos
  • Redes neuronales convolucionales para visión artificial
  • Modelos de lenguaje: GPT, Llama, Mistral
  • Algoritmos de clustering para segmentación de usuarios
CriteriosCE-aCE-b
Machine Learning (ML)
Conceptos base
▾
Subcampo de la IA que permite a los sistemas aprender y mejorar automáticamente a partir de la experiencia sin ser programados explícitamente. Los modelos se entrenan con datos etiquetados o no etiquetados.
Tipos principales
  • Supervisado: aprende con etiquetas (clasificación, regresión)
  • No supervisado: sin etiquetas (clustering, PCA)
  • Por refuerzo: aprende por recompensas (robótica, videojuegos)
CriteriosCE-aCE-bCE-e
Deep Learning
Conceptos base
▾
Subcampo del ML basado en redes neuronales artificiales con múltiples capas ocultas. Aprende representaciones jerárquicas de los datos. Requiere grandes volúmenes de datos y GPUs/TPUs.
Aplicaciones
  • Reconocimiento de voz e imagen
  • Procesamiento del lenguaje natural (NLP)
  • Generación de texto, imagen y código (IA generativa)
CriteriosCE-aCE-c
IA Generativa / LLM
Conceptos base
▾
Modelos de IA capaces de generar contenido nuevo (texto, imagen, audio, código) a partir de un prompt. Los LLM (Large Language Models) como GPT-4 o Gemini se entrenan sobre enormes corpus textuales.
Relevancia para DAW/DAM
  • GitHub Copilot: autocompletar y generar código
  • ChatGPT API: integrar conversación en apps
  • Generación automática de documentación y tests

7 términos
Big Data
Técnico
▾
Conjuntos de datos cuyo volumen, velocidad y variedad superan la capacidad de las herramientas tradicionales de procesamiento. La IA necesita Big Data para encontrar patrones de valor empresarial.
Las 5 V del Big Data
  • Volumen: terabytes o petabytes de datos
  • Velocidad: generación y procesamiento en tiempo real
  • Variedad: estructurados, semiestructurados, no estructurados
  • Veracidad: calidad y fiabilidad del dato
  • Valor: utilidad para la toma de decisiones
CriteriosCE-b
Minería de datos
Técnico
▾
Proceso de descubrimiento de patrones, correlaciones y anomalías en grandes conjuntos de datos usando técnicas estadísticas y de ML. Extrae conocimiento accionable para la empresa.
Técnicas principales
  • Clasificación: asignar categorías a nuevas instancias
  • Regresión: predecir valores numéricos continuos
  • Clustering: agrupar elementos similares sin etiquetas
  • Asociación: "quien compra X también compra Y"
CriteriosCE-b
Modelo / Entrenamiento
Técnico
▾
Un modelo de IA es una representación matemática aprendida a partir de datos. El entrenamiento es el proceso iterativo de ajuste de parámetros para minimizar el error entre predicciones y valores reales.
Ciclo de vida
  • Recolección y limpieza de datos
  • División: train / validation / test
  • Entrenamiento e iteración de hiperparámetros
  • Evaluación con métricas (accuracy, F1, RMSE…)
  • Despliegue en producción (API, SDK, edge)
CriteriosCE-aCE-b
API de IA / SDK
Técnico
▾
Mecanismo mediante el cual una aplicación consume capacidades de IA de terceros sin necesidad de entrenar modelos propios. Permite integrar NLP, visión, voz o generación de contenido con pocas líneas de código.
Servicios principales
  • OpenAI API: chat, embeddings, visión, generación
  • Google Cloud AI: Vision API, Natural Language, Translate
  • Azure Cognitive Services: Face, Speech, Text Analytics
  • TensorFlow.js: modelos en el navegador (relevante DAW)
CriteriosCE-f
Sesgo (Bias)
Técnico
▾
Error sistemático en un modelo causado por datos de entrenamiento no representativos o prejuicios históricos codificados en los datos. Un modelo sesgado puede producir decisiones injustas o discriminatorias.
Ejemplos reales
  • Sistemas de selección de CV que penalizan a mujeres
  • Reconocimiento facial con peores resultados en piel oscura
  • Algoritmos de crédito que discriminan por código postal

Automatización de procesos (RPA + IA)
Técnico
▾
La automatización de procesos robóticos (RPA) ejecuta tareas repetitivas mediante robots de software. Combinada con IA (IPA), puede tomar decisiones, interpretar documentos y adaptarse a variaciones.
Casos de uso
  • Procesamiento automático de facturas con OCR + IA
  • Respuesta automática a incidencias de soporte
  • Clasificación y enrutamiento de correos en empresas

TDH — Tecnologías Digitales Habilitadoras
Técnico
▾
Tecnologías estratégicas que actúan como base para la transformación digital. La IA se integra y se potencia al combinarse con ellas.
Relación con la IA
  • Cloud computing: entrena y despliega modelos (AWS, Azure, GCP)
  • IoT: fuente masiva de datos en tiempo real
  • Edge computing: inferencia local sin latencia
  • Ciberseguridad: detección de amenazas con IA

Ética y legal

5 términos
AI Act — Reglamento Europeo de IA
Ética · Legal
▾
Primera regulación integral de la IA a nivel mundial, aprobada por la UE en 2024. Clasifica los sistemas de IA según su nivel de riesgo y establece obligaciones proporcionales.
Clasificación de riesgos
  • Inaceptable: prohibido (puntuación social, reconocimiento facial masivo)
  • Alto: requisitos estrictos (empleo, educación, justicia)
  • Limitado: obligaciones de transparencia (chatbots, deepfakes)
  • Mínimo: sin restricciones específicas (filtros de spam)
CriteriosCE-c
RGPD / LOPDGDD
Ética · Legal
▾
El Reglamento General de Protección de Datos (UE) y la Ley Orgánica española regulan la recogida y tratamiento de datos personales. Los sistemas de IA deben cumplir principios de licitud, transparencia y seguridad.
Principios aplicados a la IA
  • Minimización: solo los datos estrictamente necesarios
  • Transparencia: informar sobre decisiones automatizadas
  • Derecho de oposición: rechazar decisiones automáticas
  • Privacidad por diseño: protección desde el diseño del sistema
CriteriosCE-bCE-c
Derechos de autor e IA
Ética · Legal
▾
El uso de contenidos protegidos para entrenar modelos o generar contenido derivado plantea conflictos legales abiertos. Hay que analizar licencias, autoría y los límites de la normativa de propiedad intelectual.
Casos relevantes
  • Demandas de artistas contra generadores de imagen (Midjourney)
  • New York Times vs. OpenAI por uso de artículos en entrenamiento
  • Autoría de obras generadas por IA: ¿quién es el titular?
CriteriosCE-c
Principios éticos de la IA
Ética · Legal
▾
Marco de valores que deben guiar el diseño, desarrollo y despliegue de sistemas de IA responsables. Los profesionales tienen responsabilidad directa sobre los sistemas que crean o integran.
Principios fundamentales
  • Explicabilidad (XAI): el modelo debe justificar sus decisiones
  • No discriminación: ausencia de sesgos que perjudiquen colectivos
  • Supervisión humana: control sobre decisiones automatizadas relevantes
  • Privacidad: protección de datos personales implicados
  • Sostenibilidad: impacto ambiental del entrenamiento de modelos
CriteriosCE-c
Decisión automatizada
Ética · Legal
▾
Resolución tomada sin intervención humana significativa por un sistema de IA. El RGPD y el AI Act regulan específicamente las decisiones automatizadas que afecten a derechos o intereses legítimos de las personas.
Ejemplos de alto impacto
  • Concesión o denegación automática de un crédito
  • Filtrado de candidatos en procesos de selección
  • Diagnóstico médico sin revisión clínica
  • Puntuación de riesgo en sistemas de justicia penal

Sectores y aplicación

6 términos
Sistema de recomendación
Sectores · Aplicación
▾
Sistema de IA que predice la preferencia de un usuario sobre ítems (productos, películas, canciones) y los sugiere de forma personalizada. Una de las aplicaciones más directamente integrables en DAW/DAM.
Enfoques
  • Filtrado colaborativo: usuarios similares tienen gustos similares
  • Basado en contenido: ítems similares a los que ya gustaron
  • Híbrido: combina ambos enfoques (Netflix, Spotify)
CriteriosCE-dCE-f
Chatbot / Asistente virtual
Sectores · Aplicación
▾
Sistema de IA que simula una conversación con el usuario mediante procesamiento del lenguaje natural (NLP). Desde chatbots simples con reglas hasta asistentes avanzados con LLMs.
Integración en DAW/DAM
  • Widget de soporte en sitios web (OpenAI API + React)
  • Asistente de navegación en apps móviles
  • Chatbot de ventas integrado en e-commerce
  • FAQ inteligente con búsqueda semántica
CriteriosCE-dCE-f
Visión artificial
Sectores · Aplicación
▾
Campo de la IA que permite a los sistemas interpretar información visual (imágenes y vídeo) usando redes neuronales convolucionales (CNN) entrenadas con grandes datasets etiquetados.
Aplicaciones por sector
  • Sanidad: detección de tumores en radiografías
  • Industria: control de calidad visual en producción
  • DAM: reconocimiento de objetos en apps móviles (Google ML Kit)
CriteriosCE-dCE-f
Mantenimiento predictivo
Sectores · Aplicación
▾
Aplicación de IA en industria para predecir fallos en maquinaria antes de que ocurran, analizando datos de sensores IoT en tiempo real. Reduce tiempos de parada y costes de reparación.
Funcionamiento
  • Sensores recogen temperatura, vibración, presión
  • El modelo identifica patrones previos a fallos históricos
  • Alerta anticipada permite programar mantenimiento preventivo
CriteriosCE-d
Detección de fraude y scoring
Sectores · Aplicación
▾
Uso de ML en el sector financiero para identificar transacciones fraudulentas en tiempo real o evaluar la solvencia de un cliente. Procesa millones de operaciones simultáneas con latencia de milisegundos.
Impacto empresarial
  • Reducción de pérdidas por fraude en tarjetas de crédito
  • Scoring automático: decisión de crédito en segundos
  • Detección de blanqueo de dinero por patrones de comportamiento

Analítica educativa y aprendizaje adaptativo
Sectores · Aplicación
▾
Aplicación de la IA al sector educativo para personalizar el itinerario de aprendizaje de cada estudiante, detectar dificultades tempranamente y proporcionar retroalimentación automática.
Casos de uso

  • Tutores virtuales que detectan lagunas conceptuales
  • Análisis de engagement en Moodle

Lenguajes y ecosistemas

5 términos
Python — Ecosistema IA
Lenguajes
▾
Lenguaje dominante en IA y ciencia de datos. Su sintaxis clara y su ecosistema de librerías especializadas lo convierten en el estándar de facto para investigación, prototipado y producción de modelos.
Librerías clave
  • NumPy / Pandas: manipulación de datos y arrays numéricos
  • Scikit-learn: ML clásico (clasificación, regresión, clustering)
  • TensorFlow / Keras: deep learning de Google
  • PyTorch: deep learning de Meta, muy usado en investigación
  • Hugging Face Transformers: LLMs, NLP, visión
CriteriosCE-e
JavaScript — IA en el navegador
Lenguajes
▾
Relevante para DAW: permite ejecutar modelos de IA directamente en el cliente sin enviar datos al servidor. TensorFlow.js permite portar modelos Python o entrenar modelos ligeros en el navegador.
Librerías
  • TensorFlow.js: visión y texto en el navegador
  • Transformers.js: modelos Hugging Face en cliente
  • LangChain.js: cadenas de prompts con LLMs en apps web
CriteriosCE-eCE-f
R — Estadística y análisis
Lenguajes
▾
Lenguaje especializado en estadística, análisis exploratorio y visualización. Muy usado en investigación, bioinformática y finanzas donde el rigor estadístico es fundamental.
Paquetes destacados
  • tidyverse: manipulación y transformación de datos
  • ggplot2: visualización de datos de alta calidad
  • tidymodels / caret: ML con enfoque estadístico
CriteriosCE-e
Java — IA en sistemas empresariales
Lenguajes
▾
Usado en entornos empresariales de gran escala. Permite integrar capacidades de ML en sistemas Java existentes sin migrar toda la infraestructura.
Librerías
  • Weka: suite clásica de ML con interfaz gráfica y API
  • Deeplearning4j: deep learning en JVM con Hadoop/Spark
  • Apache Spark MLlib: ML distribuido para Big Data
CriteriosCE-e
Julia y C/C++ — Alto rendimiento
Lenguajes
▾
Julia combina la facilidad de Python con el rendimiento de C, ideal para computación científica. C/C++ es la base de las librerías de alto rendimiento y se usa para inferencia en edge y sistemas embebidos.
Cuándo se usan
  • Julia: investigación científica, modelos financieros de alta precisión
  • C/C++: núcleo de TensorFlow, PyTorch y OpenCV
  • C++ con libtorch: despliegue de modelos PyTorch en producción embebida
