Glosario rápido (DAW 1º)

Runtime
Entorno de ejecución de una app (p. ej., Node.js, JVM, PHP-FPM).
Escalado
Ajuste de recursos (CPU/RAM/instancias) para soportar más o menos carga.
Latencia
Tiempo de respuesta percibido; clave en apps interactivas o en tiempo real.
CDN
Red de distribución de contenidos que acerca archivos a los usuarios (edge).

Diccionario evaluable

Cloud Computing (nube)

Definición
Modelo de acceso por red, bajo demanda, a un conjunto compartido de recursos configurables que se aprovisionan/liberan rápidamente con mínima gestión. Marco: NIST
Niveles (modelos de servicio)
  • IaaS Infraestructura: VMs, red, discos; el equipo administra SO, runtime y despliegue.
  • PaaS Plataforma: runtime/hosting gestionado; el equipo se centra en código y configuración.
  • SaaS Software: aplicación lista para usar; el usuario configura y utiliza.
Indicadores evaluables
  • Clasifica servicios reales como IaaS/PaaS/SaaS justificando responsabilidad del proveedor vs equipo.
  • Diferencia “cloud ≠ internet” (la nube es un modelo de servicio; internet es la red).

Funciones principales de la cloud

Definición
Capacidades típicas que la nube ofrece para construir, ejecutar y operar aplicaciones web.
Funciones (mínimo evaluable)
  • Procesamiento de datos: backend, colas, tareas programadas (jobs).
  • Intercambio de información: APIs REST/GraphQL, mensajería, sincronización.
  • Ejecución de aplicaciones: hosting, balanceo, autoescalado.
  • Almacenamiento: ficheros/objetos, BBDD, copias de seguridad.
